## 1. 프로젝트 목적

### 배경

한국 개인 투자자(일명 **개미**)들 사이에서는 오랫동안 "오늘 일진이 좋으면 주식도 잘 되더라"는 말이 회자되어 왔습니다. 별자리 운세·사주는 MZ세대를 중심으로 여전히 높은 콘텐츠 소비력을 가진 엔터테인먼트 영역입니다.

**개미의 하루**는 이 두 문화를 결합하여:

- 📊 동양 전통 역술(사주명리·만세력)의 **오행(五行) 분석**
- ⭐ 서양 점성술의 **별자리 운세**
- 📈 국내(KOSPI·KOSDAQ) / 미국(NYSE·NASDAQ) **주식 섹터 추천**

을 하나의 재미있는 엔터테인먼트 앱으로 엮어냅니다.

### 목표

| 목표 | 설명 |
|------|------|
| **엔터테인먼트** | 딱딱한 주식 정보를 운세라는 친숙한 포맷으로 재해석 |
| **완전 로컬** | 외부 API 없이 브라우저 내에서 모든 계산 수행 |
| **교육적 가치** | 사주명리학의 오행 개념을 현대적 맥락으로 소개 |
| **접근성** | 설치 없이 브라우저에서 즉시 실행 가능한 React 앱 |

> ⚠️ **본 앱은 순수 엔터테인먼트 목적이며, 실제 투자 조언이 아닙니다.**

### 3-1. ⭐ 퀵 모드 — 별자리 운세

생년월일의 **월/일만** 입력하면 즉시 결과를 확인할 수 있는 빠른 모드입니다.

```
입력: 생일 월 + 일 (예: 3월 25일)
  ↓
별자리 자동 판별 (예: 양자리 ♈)
  ↓
오늘 일진(日辰) 계산
  ↓
별자리 원소 + 일진 오행 조합 → 투자 스탠스 결정
  ↓
날짜 시드 기반 템플릿 풀에서 운세 문구 선택
```

### 3-2. ☯️ 딥 모드 — 사주 오행 심층 분석

생년월일, 태어난 시간, 성별을 입력하여 **사주팔자(四柱八字)**를 완전 로컬 계산합니다.

```
입력: 생년월일 + 시주(時柱) + 성별
  ↓
만세력 알고리즘으로 연주·월주·일주·시주 계산
  ↓
천간(天干) + 지지(地支) + 지장간(地藏干) → 오행 점수 산출
  ↓
부족한 오행(最弱) 파악 → 해당 오행을 보충하는 섹터 추천
  ↓
오늘 일진 오행과의 상생·상극 관계 → 투자 스탠스 결정
```

**사주팔자 화면 예시:**

```
연주(年柱)   월주(月柱)   일주(日柱)   시주(時柱)
  甲子         丙寅         庚午         壬申
 (갑자)       (병인)       (경오)       (임신)
  木·水        火·木        金·火        水·金
```

### 3-3. 📅 오늘의 일진 (日辰) 실시간 표시

앱 상단에 **오늘 날짜의 일진**이 항상 표시됩니다.

- 60갑자(六十甲子) 순환 계산
- 천간(天干)의 오행 · 지지(地支)의 오행 실시간 표시
- 해당 일의 동물 띠 표시 (예: "용의 날")

---

### 3-4. 📊 투자 스탠스 결정 엔진

오행의 **상생(相生)** · **상극(相剋)** 원리를 활용한 3단계 스탠스:

```
🔴 적극 매수 (STRONG BUY)
   — 일진 천간이 火·木 기운 + 태어난 오행과 상생 관계일 때

⚖️ 관망 (HOLD)
   — 일진이 土 기운 또는 명확한 상승·하락 신호 없을 때

🔵 매도 후 현금 확보 (SELL)
   — 일진 천간이 金·水 기운 + 태어난 오행과 상극 관계일 때
```

**오행 상생·상극 관계:**

```
상생(相生, 서로 도움):  木 → 火 → 土 → 金 → 水 → 木
상극(相剋, 서로 억제):  木 → 土 → 水 → 火 → 金 → 木
```

### 3-5. 🏦 국장·미장 섹터 추천

부족한 오행 또는 별자리 원소에 맞는 **국내·미국 주식 섹터**를 각각 추천합니다.

| 오행 | 국장 추천 섹터 | 미장 추천 섹터 |
|------|--------------|--------------|
| 木 (목) | 바이오·헬스케어, 친환경·ESG | 헬스케어 ETF, 그린에너지 |
| 火 (화) | 반도체·전자, AI·로봇 | 빅테크 (M7), AI 반도체 |
| 土 (토) | 건설·부동산, 인프라·유틸리티 | 부동산 ETF, 인프라 ETF |
| 金 (금) | 철강·비철금속, 방산·항공우주 | 금·원자재 ETF, 방산 ETF |
| 水 (수) | 금융·은행, 해운·물류 | 금융 ETF, 물류·해운 |

- ☀️ **낮의 장 (KR)**: KOSPI·KOSDAQ 단기 테마 중심
- 🌙 **밤의 장 (US)**: NYSE·NASDAQ 글로벌 우량주 중심

```javascript
// 날짜를 시드로 사용 → 같은 날은 항상 같은 결과, 매일 다른 문구
function dateSeed(date, offset = 0) {
  return date.getFullYear() * 10000
    + (date.getMonth()+1) * 100
    + date.getDate() + offset;
}
function pickTemplate(arr, seed) {
  return arr[((seed % arr.length) + arr.length) % arr.length];
}
```

## 4. 알고리즘 상세

### 4-1. 일진 계산 (핵심 알고리즘)

```javascript
// 기준: 1900년 1월 1일 = 甲辰일 (60갑자 40번째, 0-indexed: 39)
const EPOCH = new Date(1900, 0, 1);

function getIljin(date) {
  const days = Math.floor((date - EPOCH) / 86400000);
  const idx  = ((40 + days) % 60 + 60) % 60;  // 40 = 甲辰의 순번
  const g    = idx % 10;   // 천간 인덱스 (0~9)
  const z    = idx % 12;   // 지지 인덱스 (0~11)
  return {
    gan:   ["甲","乙","丙","丁","戊","己","庚","辛","壬","癸"][g],
    zhi:   ["子","丑","寅","卯","辰","巳","午","未","申","酉","戌","亥"][z],
    ganEl: ["wood","wood","fire","fire","earth","earth","metal","metal","water","water"][g],
    zhiEl: ["water","earth","wood","wood","earth","fire","fire","earth","metal","metal","earth","water"][z],
  };
}
```

### 4-2. 연주 계산

```javascript
function getYearPillar(year, month, day) {
  // 입춘(立春, 2월 4일) 기준으로 연주 전환
  const y = (month === 1 || (month === 2 && day < 4)) ? year - 1 : year;
  const g = ((y - 4) % 10 + 10) % 10;   // 천간: 갑(甲)년 기준
  const z = ((y - 4) % 12 + 12) % 12;   // 지지: 자(子)년 기준
  return { gan: GAN[g], zhi: ZHI[z], ... };
}
```

### 4-3. 월주 계산 (오기법 적용)

### 4-4. 오행 점수 산출

```javascript
function calcFiveElements(year, month, day, hourInput) {
  const scores = { wood:0, fire:0, earth:0, metal:0, water:0 };

  for (const pillar of [yearPillar, monthPillar, dayPillar, hourPillar]) {
    scores[pillar.ganEl] += 15;  // 천간 가중치
    scores[pillar.zhiEl] += 10;  // 지지 가중치
  }

  // 지장간(地藏干) 간략 반영: 진술축미(辰戌丑未)는 土 추가
  for (const p of pillars) {
    if ([1, 4, 7, 10].includes(p.zhiIdx)) scores.earth += 5;
  }

  // 정규화 (백분율 변환)
  const total = Object.values(scores).reduce((a,b) => a+b, 0);
  return Object.fromEntries(
    Object.entries(scores).map(([k,v]) => [k, Math.round(v/total*100)])
  );
}
```

## 8. 이슈 및 해결 과정

### 이슈 1: LLM API 직접 호출 시 CORS 에러 및 보안 취약점

**문제 상황**

Vercel 배포 후, 클라이언트(React)에서 Anthropic API를 직접 호출하자 브라우저 콘솔에 `CORS (Cross-Origin Resource Sharing)` 에러가 발생하며 데이터가 로드되지 않았습니다.

**원인 분석**

- **보안 정책:** 브라우저는 보안상의 이유로 스크립트 내에서 타 도메인(Anthropic API)으로의 직접적인 리소스 요청을 차단합니다.
- **키 노출 위험:** 프론트엔드 코드에 API Key를 포함시킬 경우, 네트워크 탭을 통해 키가 외부로 유출될 치명적인 보안 위험이 존재합니다.

**해결 방안**

아키텍처를 클라이언트 직접 호출 방식에서 **Backend Proxy** 경유 방식으로 변경했습니다. Vercel의 Serverless Functions를 활용하여 API Key를 서버 환경 변수(`Environment Variable`)로 안전하게 관리하고, 서버 측에서 API를 호출한 뒤 결과만 클라이언트로 전달하도록 수정했습니다.

```
[Before]  Client (React) → Anthropic API  ← CORS 차단, 키 노출 위험
[After]   Client (React) → Vercel Function → Anthropic API  ← 안전
```

---

### 이슈 2: 오행 데이터 시각화의 정밀도 문제

**문제 상황**

초기에 오행(목, 화, 토, 금, 수)의 수치를 단순 랜덤값이나 텍스트로만 처리하여 사용자에게 직관적인 정보를 제공하지 못했습니다.

**해결 방안**

각 오행의 수치를 0~100 사이의 값으로 정규화하고, CSS `transition` 속성이 적용된 프로그레스 바(Progress Bar) 컴포넌트로 시각화했습니다. 이를 통해 사용자가 자신의 '부족한 기운'을 한눈에 파악할 수 있도록 개선했습니다.
